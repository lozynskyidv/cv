### **File Types:**

| File | Purpose | Who Edits | When |
|------|---------|-----------|------|

- Contains raw facts from LinkedIn (no pre-written bullets)
- AI reads this and generates compelling "Accomplished X by Y by Z" bullets
- AI makes reasonable metric estimates (you correct in visual review if needed)
- Minimal maintenance - just facts

## ğŸ¯ How the AI Workflow Works

When you paste a job URL in a **new chat**, the AI automatically:

1. **Loads `.cursorrules`** (auto-configuration for the chat)
2. **Reads `TAILORING_GUIDE.md`** (AI's instruction manual)
3. **Reads `cv_master.yaml`** (your source of truth - never modified)
4. **Analyzes the job posting** (requirements, keywords, role type)
5. **Asks 3-5 strategic questions:**
   - "Should I lead with founder or growth story?"
   - "Emphasize technical or business side?"
   - "Which metrics matter most?"
6. **Creates `cv_[job_name].yaml`** (copies from master)
7. **Edits the job-specific file** (master untouched)
8. **Generates PDF** via `generate.py`
9. **Opens PDF** in Preview app automatically
10. **You review** the formatted CV visually
11. **You approve or request changes**

## ğŸ”§ Troubleshooting

### **AI doesn't recognize the system in new chat:**
Make sure you're in the `/cv` folder when opening the chat. If still not working, explicitly reference:

### **PDF generation fails:**
```bash

# If missing, reinstall
./setup.sh
```

### **CV looks wrong:**
```bash
# Check the generated source file
cat output/cv_Product_Manager_Google.typ

# Verify YAML is valid
python3 -c "import yaml; yaml.safe_load(open('cv_master.yaml'))"
```

### **Placeholder metrics accumulating:**
Every 10 applications, search for `[VERIFY:` in your YAML files and update with real numbers.
